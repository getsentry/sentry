# Color

## Categories

Colors have been organized into separate scales for specific use cases.

| Category     | Description                                                                                            |
| ------------ | ------------------------------------------------------------------------------------------------------ |
| `background` | Painting an elements background.                                                                       |
| `content`    | Painting text and icons which need to stand out the most.                                              |
| `graphics`   | Painting icons and other graphical elements which donâ€™t need to stand out as much as `content` tokens. |
| `borders`    | Painting borders of containers or visual diagrams.                                                     |
| `shadows`    | Painting shadows of containers.                                                                        |
| `controls`   | Painting interactive elements and all their states.                                                    |
| `syntax`     | Painting `<code/>` elements that depend on PrismJS.                                                    |
| `focus`      | Painting `:focus` or validation states.                                                                |
| `dataviz`    | Painting chart data.                                                                                   |

## Variants

For consistency, token scales and `variant` properties have been standardized with the following terminology.

| Keyword       | Description                                                                                                         |
| ------------- | ------------------------------------------------------------------------------------------------------------------- |
| `accent`      | The call-to-action colour for interactive elements.                                                                 |
| `info`        | Semantically charged, informing users of important facts.                                                           |
| `promotion`   | Semantically charged, encouraging users to invest in Sentry.                                                        |
| `warning`     | Semantically charged, alerting users of a potential issue.                                                          |
| `danger`      | Semantically charged, alerting users of a issue.                                                                    |
| `success`     | Semantically charged, informing users of a desired end-state.                                                       |
| `transparent` | Can be visually equivalent alternatives to related tokens or the name of a style variant.                           |
| `chonky`      | A style variant for elements that are visually protruding or depressed relative to the surface the element sits on. |

export function Space() {
  const theme = useTheme();
  return (
    <Storybook.TokenReference
      scale="space"
      tokens={theme.space}
      renderToken={({value}) => (
        <Flex
          align="center"
          justify="center"
          as="div"
          width={value}
          height="16px"
          borderLeft="accent"
          borderRight="accent"
          style={{boxSizing: 'border-box'}}
        >
          <Container
            as="div"
            width="100%"
            height="1px"
            borderTop="accent"
            style={{boxSizing: 'border-box'}}
          />
        </Flex>
      )}
    />
  );
}

export function Radius() {
  const theme = useTheme();
  return (
    <Storybook.TokenReference
      scale="radius"
      tokens={theme.isChonk ? theme.radius : {md: '6px'}}
      renderToken={({token}) => (
        <Container
          as="div"
          style={{
            boxSizing: 'border-box',
            width: '48px',
            height: '48px',
            background: color(theme.tokens.border.accent).alpha(0.2).string(),
          }}
          border="accent"
          radius={token}
        />
      )}
    />
  );
}

export function FontSize() {
  const theme = useTheme();
  return (
    <Storybook.TokenReference
      scale="fontSize"
      tokens={theme.fontSize}
      renderToken={({token}) => (
        <Text size={token} variant="accent">
          Aa
        </Text>
      )}
    />
  );
}

export function FontWeight() {
  const theme = useTheme();
  return (
    <Storybook.TokenReference
      scale="fontWeight"
      tokens={theme.fontWeight}
      renderToken={({value}) => (
        <Text size="lg" style={{fontWeight: value}} variant="accent">
          {value}
        </Text>
      )}
    />
  );
}
