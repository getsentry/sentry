`<Slider>` is a range slider component that allows users to select a numeric value by dragging a handle along a track. It provides visual and interactive feedback, making it ideal for adjusting continuous values.

Use `<Slider>` for selecting numeric values from a range, such as volume controls, zoom levels, or adjustable thresholds.

```jsx
const [value, setValue] = useState(50);

<Slider value={value} onChange={setValue} min={0} max={100} />;
```

## Basic Usage

The simplest slider with default range (0-100):

export function BasicDemo() {
  const [value, setValue] = useState(50);
  return (
    <Stack as="label" align="center" gap="md">
      Basic slider (value: {value})
      <Slider value={value} onChange={setValue} />
    </Stack>
  );
}

## Step Control

The `step` prop restricts which values can be selected. This creates discrete stops along the slider track.

export function StepDemo() {
  const [value, setValue] = useState(0);
  return (

        Steps of 25 (value: {value})

## Controlled vs Uncontrolled

Sliders can be controlled or uncontrolled:

```jsx
// Controlled (recommended)
const [value, setValue] = useState(50);
<Slider value={value} onChange={setValue} />

// Uncontrolled
<Slider defaultValue={50} />
```

## Keyboard Controls

`<Slider>` provides full keyboard support:

- **Arrow Left/Down**: Decrease value by step
- **Arrow Right/Up**: Increase value by step
- **Home**: Jump to minimum value
- **End**: Jump to maximum value
- **Page Up/Down**: Larger increments (10Ã— step)

## Usage Patterns

### Volume Controls

```jsx
const [volume, setVolume] = useState(75);

<label>
  Volume
  <Slider
    value={volume}

    min={0}
```

## Accessibility

`<Slider>` follows the [WAI-ARIA Slider pattern](https://www.w3.org/WAI/ARIA/apg/patterns/slider/) and meets WCAG 2.2 AA standards:

- [2.1.1 Keyboard](https://www.w3.org/TR/WCAG22/#keyboard): Full keyboard control
- [2.4.7 Focus Visible](https://www.w3.org/TR/WCAG22/#focus-visible): Clear focus indicators

The component automatically includes:

- `role="slider"`
- `aria-valuemin`, `aria-valuemax`, `aria-valuenow` attributes
- `aria-valuetext` for formatted values
- Keyboard navigation

### Developer Responsibilities

**Labels**

- Every slider must have an associated label
- Use `<label>`, `aria-label`, or `aria-labelledby`

```jsx
// Good: Visible label
<label>
  Volume
  <Slider value={volume} onChange={setVolume} />
</label>

// Good: aria-label
<Slider value={volume} onChange={setVolume} aria-label="Volume level" />
```

**Value Communication**

- Use `formatLabel` to provide meaningful value displays
- Consider showing the current value near the slider
- Ensure the value format is clear to users

**Keyboard Interaction**

- Test that all keyboard shortcuts work correctly
- Ensure focus indicators are visible
- Verify that Home/End keys reach min/max values

For more information, see the [WAI-ARIA Slider practices](https://www.w3.org/WAI/ARIA/apg/patterns/slider/).
