The `StatusIndicator` is a small circular indicator that communicates status or severity. It supports three semantic variants mapped to design tokens.

Placement is the caller's responsibility — the component renders as an inline element with no positioning of its own.

## Accessibility

The component handles three patterns based on the props provided:

| Props                 | Behaviour                                                       |
| --------------------- | --------------------------------------------------------------- |
| No `aria-label`       | `aria-hidden="true"` — decorative, invisible to screen readers  |
| `aria-label` only     | `role="img"` + `aria-label` — standalone meaningful indicator   |
| `role` + `aria-label` | Explicit role respected (e.g. `role="status"` for live updates) |

Omit `aria-label` when the indicator accompanies visible text that already conveys the same meaning.

```jsx
// Decorative — the adjacent text already communicates the status
<Flex align="center" gap="xs">
  <StatusIndicator variant="danger" />
  <Text>Critical error</Text>
</Flex>

// Meaningful — the indicator is the only signal
<StatusIndicator variant="warning" aria-label="Warnings present" />

// Live region — updates are announced to screen readers
<StatusIndicator variant="danger" role="status" aria-label="2 new errors" />
```
