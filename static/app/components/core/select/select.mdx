`<Select>` is a dropdown component for choosing one or multiple items from a list of options. It supports searching, clearing selections, custom option rendering with icons, and both single and multi-select modes.

Use `<Select>` for form fields where users need to choose from a predefined set of options. For more complex selection scenarios with sections or custom triggers, consider `<CompactSelect>` instead.

```jsx
<Select
  placeholder="Choose an option"
  options={[
    {value: 'option1', label: 'Option 1'},
    {value: 'option2', label: 'Option 2'},
    {value: 'option3', label: 'Option 3'},
  ]}
/>
```

## Usage Patterns

### Form Fields

Pair selects with labels for accessible forms:

```jsx
<label htmlFor="environment">
  Environment
  <Select
    id="environment"
    placeholder="Choose an environment"
    options={[
      {value: 'prod', label: 'Production'},
      {value: 'staging', label: 'Staging'},
      {value: 'dev', label: 'Development'},
    ]}
  />
</label>
```

### Controlled Selects

For controlled selects, manage the value in React state:

```jsx
const [value, setValue] = useState(null);

<Select value={value} onChange={setValue} options={options} />;
```

### Multi-Select for Tags

Use multi-select for tag or category selection:

```jsx
<label>
  Tags
  <Select
    multiple
    isSearchable
    isClearable
    placeholder="Add tags"
    value={selectedTags}
    onChange={setSelectedTags}
    options={allTags}
  />
</label>
```

## Accessibility

`<Select>` is built with accessibility in mind and meets WCAG 2.2 AA standards:

- [2.1.1 Keyboard](https://www.w3.org/TR/WCAG22/#keyboard): Full keyboard navigation with arrow keys, Enter, Escape
- [2.4.7 Focus Visible](https://www.w3.org/TR/WCAG22/#focus-visible): Clear focus indicators

The component follows the [WAI-ARIA Combobox](https://www.w3.org/WAI/ARIA/apg/patterns/combobox/) and [Listbox](https://www.w3.org/WAI/ARIA/apg/patterns/listbox/) patterns, including:

- `role="combobox"` on the input
- `aria-expanded` state management
- `aria-controls` linking to the options list
- `aria-activedescendant` for keyboard navigation
- Screen reader announcements for selection changes

### Developer Responsibilities

**Labels** ([WCAG 3.3.2](https://www.w3.org/TR/WCAG22/#labels-or-instructions))

- Every select must have an associated label
- Use a `<label>` element with `htmlFor`, wrap the select in a `<label>`, or provide `aria-label`
- In-field labels (`inFieldLabel`) supplement but don't replace proper labeling

```jsx
// Good: Visible label with htmlFor
<label htmlFor="project">Project</label>
<Select id="project" options={projects} />

// Good: Wrapping label
<label>
  Project
  <Select options={projects} />
</label>

// Good: aria-label when context is clear
<Select aria-label="Filter by project" options={projects} />
```

**Keyboard Navigation**

- **Arrow Up/Down**: Navigate options
- **Enter/Space**: Select option or open dropdown
- **Escape**: Close dropdown
- **Type to search**: When `isSearchable` is enabled

**Required Fields**

- Use the `required` prop for required fields
- Indicate visually in the label (e.g., with an asterisk)
- Provide clear validation feedback

**Error Handling**

- Use `aria-invalid` and `aria-describedby` for validation errors
- Provide clear, specific error messages

```jsx
<Select
  id="project"
  options={projects}
  aria-invalid={hasError}
  aria-describedby={hasError ? 'project-error' : undefined}
/>;
{
  hasError && (
    <span id="project-error" role="alert">
      Please select a project
    </span>
  );
}
```

For more information, see the [WAI-ARIA Combobox practices](https://www.w3.org/WAI/ARIA/apg/patterns/combobox/).
