The `Link` component provides a unified API for all navigation, handling both internal routes and external URLs through a single `href` prop. External URLs are automatically detected and rendered with appropriate security attributes.

## Basic Usage

Pass any URL to the `href` prop. Internal paths are routed through React Router, while external URLs (`http://` or `https://`) render as plain `<a>` elements with `target="_blank"` and `rel="noreferrer noopener"`.

<Storybook.Demo>
  <Storybook.SideBySide vertical>
    <Link href="/organizations/sentry/issues/">Internal: Issues page</Link>
    <Link href="https://docs.sentry.io">External: Sentry Docs</Link>
  </Storybook.SideBySide>
</Storybook.Demo>
```jsx
<Link href="/organizations/sentry/issues/">Internal: Issues page</Link>
<Link href="https://docs.sentry.io">External: Sentry Docs</Link>
```

## Link Target Format

The `href` prop accepts either a string path or a `LocationDescriptor` object. When using literal paths, always use slug-based URLs to ensure compatibility across different deployment environments (customer domains and single-tenant).

<Storybook.Demo>
  <Storybook.SideBySide vertical>
    <Link href="/organizations/sentry/issues/">String path</Link>
    <Link href={{pathname: '/settings/account/'}}>LocationDescriptor object</Link>
  </Storybook.SideBySide>
</Storybook.Demo>
```jsx
<Link href="/organizations/sentry/issues/">String path</Link>
<Link href={{pathname: '/settings/account/'}}>LocationDescriptor object</Link>
```

## External Links

External URLs are automatically detected by the `href` prop. By default, they open in a new tab with `target="_blank"` and `rel="noreferrer noopener"`.

<Storybook.Demo>
  <Link href="https://docs.sentry.io">Visit Sentry Docs</Link>
</Storybook.Demo>
```jsx
<Link href="https://docs.sentry.io">Visit Sentry Docs</Link>
```

### Opening in Same Tab

Set `openInNewTab={false}` to open external links in the same tab.

<Storybook.Demo>
  <Storybook.SideBySide vertical>
    <Link href="https://docs.sentry.io">Opens in new tab (default)</Link>
    <Link href="https://docs.sentry.io" openInNewTab={false}>
      Opens in same tab
    </Link>
  </Storybook.SideBySide>
</Storybook.Demo>
```jsx
<Link href="https://docs.sentry.io">Opens in new tab (default)</Link>
<Link href="https://docs.sentry.io" openInNewTab={false}>
  Opens in same tab
</Link>
```

## Disabled Links

Use the `disabled` prop to disable link navigation while maintaining visual presence.

<Storybook.Demo>

    <Link href="/organizations/sentry/issues/" disabled>
      Disabled link
    </Link>
  </Storybook.SideBySide>
</Storybook.Demo>
```jsx
<Link href="/issues/">Active link</Link>
<Link href="/issues/" disabled>
  Disabled Link
</Link>
```

## Migration from `ExternalLink` and `to`

The `ExternalLink` component and the `to` prop are deprecated. Use `href` for all links.

```jsx
// Before
<Link to="/issues/">Issues</Link>
<ExternalLink href="https://docs.sentry.io">Docs</ExternalLink>

// After
<Link href="/issues/">Issues</Link>
<Link href="https://docs.sentry.io">Docs</Link>
```
