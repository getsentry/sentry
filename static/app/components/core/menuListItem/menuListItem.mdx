`<MenuListItem>` is a versatile building block for creating menu items, dropdown options, and selectable list items. It provides consistent styling, interaction states, and support for icons, labels, and supplementary details.

Use `<MenuListItem>` when building custom menus, dropdown options, or selectable lists. For standard dropdowns, prefer higher-level components like `<CompactSelect>` which use `<MenuListItem>` internally.

```jsx

```

## States

`<MenuListItem>` supports several visual states:

- **isFocused**: Highlights the item when keyboard focused
- **isSelected**: Indicates the item is currently selected
- **disabled**: Prevents interaction

## Leading Items

Use `leadingItems` to add icons, checkmarks, or other elements before the label. The prop accepts a function that receives the item's state:

```jsx
<MenuListItem
  label="Selected Item"
  leadingItems={state => (state.isSelected ? <IconCheckmark /> : null)}
  isSelected
/>

<MenuListItem
  label="Item with Icon"
  leadingItems={() => <IconDocument />}
/>
```

## Usage Patterns

### Custom Dropdown Menus

```jsx
const CustomMenu = () => {
  const [selected, setSelected] = useState('option1');

  return (
    <div role="menu">
      {options.map(option => (
        <MenuListItem
          key={option.value}
          label={option.label}
          isSelected={selected === option.value}
          priority={selected === option.value ? 'primary' : 'default'}
          leadingItems={state => (state.isSelected ? <IconCheck /> : null)}
          onClick={() => setSelected(option.value)}
        />
      ))}
    </div>
  );
};
```

### Selectable Lists

```jsx
{
```

## Accessibility

`<MenuListItem>` is designed as a building block and adapts to different accessibility patterns:

- [2.1.1 Keyboard](https://www.w3.org/TR/WCAG22/#keyboard): Keyboard accessible when used in proper context
- [2.4.7 Focus Visible](https://www.w3.org/TR/WCAG22/#focus-visible): Visual feedback for focused state

### Developer Responsibilities

**Semantic Context**

- Wrap items in appropriate containers (`role="menu"`, `role="listbox"`, `role="list"`)
- Add proper role to items based on context (`role="menuitem"`, `role="option"`)
- Ensure keyboard navigation is implemented

**Labels**

- The `label` prop provides the primary text
- Additional context can be provided via `details`
- Use `aria-label` if the label doesn't fully describe the action

**Keyboard Navigation**

- Implement arrow key navigation for menu/list contexts
- Support Enter/Space for selection
- Manage focus states appropriately

For more information, see the [WAI-ARIA Menu practices](https://www.w3.org/WAI/ARIA/apg/patterns/menu/).
